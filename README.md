# Topic 12: Modules and Files

**Week 6 | CIS Intro to Programming**

---

## Overview

This week you will practice working with Python modules and file I/O — skills that connect your programs to the real world. Your program should read from or write to a file, and make use of at least one standard library module.

## Assignment Prompt

Write a Python program that works with files and uses at least one Python standard library module. Your program should:

- Read from a file, write to a file, or both
- Use at least one standard library module (e.g., `os`, `csv`, `json`, `random`, `datetime`, `math`)
- Have a clear purpose — the file interaction should be meaningful, not just for demonstration
- Include your test file(s) in the repository alongside your solution

Some ideas: a program that logs entries to a text file, a CSV reader that summarizes data, a JSON-based contact list, a program that generates a report and saves it, or a random quote picker that reads from a text file.

## Peer Review Prompt

When reviewing a classmate's code this week, consider:

- Does their program handle the case where the file doesn't exist yet?
- Did they close their file properly, or use a `with` statement?
- Is the module they chose a good fit for what they were trying to do?
- Can you run their program without needing to create your own test data?

## Submission Instructions

1. Fork this repository into your own GitHub account.
2. Write your code in a file named `solution.py` and commit it to your fork.
3. **Include any data files your program needs** (e.g., a sample `.txt` or `.csv` file) so reviewers can run your program without creating their own.
4. Submit your repo link using the **Submit Your Repo Link** form in Blackboard.
5. Open the **Class Repo Directory** in Blackboard and find two classmates' repos.
6. Leave a GitHub Issue on each with substantive feedback using the peer review prompt above as your guide.
